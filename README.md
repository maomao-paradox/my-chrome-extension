# 🚀 Manteia Boost - 智能浏览器增强解决方案

## 📋 项目概述

Manteia Boost 是一款基于 Chrome 扩展技术的企业级浏览器增强平台，为各类内网系统提供无侵入式的功能扩展与体验升级。通过模块化架构与智能技术，显著提升工作效率、优化用户体验并保障系统安全。

## 🏆 核心价值

### 业务价值
- **效率提升**：表单自动化、批量操作等功能减少 60% 手动操作时间
- **数据质量**：智能校验与规则引擎确保数据录入准确性
- **用户体验**：现代化界面与交互设计，降低系统学习成本
- **安全合规**：Shadow DOM 隔离技术确保系统稳定与数据安全
- **扩展性强**：模块化架构支持快速适配新系统与功能迭代

### 技术价值
- **架构创新**：采用分层架构与事件驱动设计，实现组件化开发
- **技术栈领先**：Vue 3 + TypeScript + Chrome Extension Manifest V3
- **性能优化**：Shadow DOM 隔离渲染，零样式冲突与脚本污染
- **安全可靠**：遵循最小权限原则，敏感数据加密存储
- **可维护性**：标准化代码规范与完善的类型定义

## 🤖 DeepSeek 独立 API 服务

项目现在额外提供了一个不依赖扩展程序、Service Worker 或内容脚本的 DeepSeek 独立服务。它会在 Node 进程内自行完成：

- DeepSeek 会话持久化
- POW 挑战计算（复用仓库里的 `wasm` 资产）
- 流式对话转发
- 会话清理

### 启动方式

```bash
npm run serve:deepseek-service
```

默认监听地址：

```text
http://127.0.0.1:8787
```

### 主要接口

1. 健康检查：`GET /health`
2. 流式/非流式对话：`POST /api/deepseek/chat`
3. 清空指定角色会话：`POST /api/deepseek/session/clear`

### 请求示例

非流式：

```bash
curl -X POST http://127.0.0.1:8787/api/deepseek/chat \
  -H "Content-Type: application/json" \
  -d '{"prompt":"介绍一下当前服务","role":"demo","stream":false}'
```

流式：

```bash
curl -N -X POST http://127.0.0.1:8787/api/deepseek/chat \
  -H "Content-Type: application/json" \
  -d '{"prompt":"流式返回一段文本","role":"demo","stream":true}'
```

清理会话：

```bash
curl -X POST http://127.0.0.1:8787/api/deepseek/session/clear \
  -H "Content-Type: application/json" \
  -d '{"role":"demo"}'
```

### 环境变量

- `DEEPSEEK_SERVICE_HOST`：服务监听地址
- `DEEPSEEK_SERVICE_PORT`：服务监听端口
- `DEEPSEEK_SESSION_FILE`：会话持久化文件路径
- `DEEPSEEK_API_BASE_URL`：DeepSeek API 基础地址
- `DEEPSEEK_AUTH_TOKEN`：认证 Token
- `DEEPSEEK_COOKIES`：请求 Cookies
- `DEEPSEEK_USER_AGENT`：请求 User-Agent
- `DEEPSEEK_CREATE_SESSION_COOKIE`：创建会话时附加的 Cookie 片段

## 🔧 技术实现亮点

### 1. 隔离渲染技术
- **Shadow DOM 应用**：所有前端组件运行在隔离树中，实现零样式冲突、零脚本污染
- **样式隔离**：组件样式完全独立，不影响目标系统
- **脚本隔离**：避免与目标系统 JavaScript 冲突

### 2. 模块化设计
- **插件系统**：功能模块化封装，支持按需加载
- **事件总线**：全局事件管理，组件间通信高效可靠
- **依赖注入**：服务层统一管理，提高代码复用性

### 3. 性能优化
- **代码分割**：按功能模块分割，减少初始加载时间
- **懒加载**：非核心功能按需加载
- **DOM 操作优化**：批量处理与虚拟 DOM 应用

### 4. 安全保障
- **权限管理**：仅申请必要的浏览器权限
- **数据加密**：敏感数据使用 AES 加密存储
- **输入验证**：严格的用户输入校验
- **最小权限原则**：各模块仅访问必要的 API

## 📖 使用指南

### 开发模式

```bash
npm install
npm run dev
```

首次启动后打开 Chrome 的 `chrome://extensions`，开启「开发者模式」，点击「加载已解压的扩展程序」，选择项目根目录下的 `dist` 文件夹。后续保持 `npm run dev` 运行即可，普通页面、弹窗、选项页、侧边栏和内容脚本改动会走 Vite/CRXJS 热更新。

需要单独调试 options 普通页面时使用：

```bash
npm run dev:options
```

正式构建仍使用：

```bash
npm run build
```

同步构建产物到本地浏览器加载目录的脚本保持不变：

```bash
npm run build-dev
```

注意：`manifest.config.ts`、Service Worker 入口、content script 注册逻辑这类扩展结构变更通常需要 Chrome 重新加载扩展；如果页面里已经注入了旧 content script，刷新目标页面后才能拿到新逻辑。
